# 🧠 Athena 持续进化机制

_创建时间：2026-04-17_

---

## 1. 学习日志系统（实时）

| 触发条件 | 记录位置 |
|----------|----------|
| 命令/操作失败 | `.learnings/ERRORS.md` |
| 用户纠正我 | `.learnings/LEARNINGS.md` (correction) |
| 发现更好方法 | `.learnings/LEARNINGS.md` (best_practice) |
| 知识过时 | `.learnings/LEARNINGS.md` (knowledge_gap) |
| 缺失功能 | `.learnings/FEATURE_REQUESTS.md` |

**规则**：每次犯错或学习，立即记录。不拖延。

---

## 2. 定期进化循环（心跳驱动）

### 每日（通过心跳）
- 检查 `.learnings/` 中是否有未处理的条目
- 如果有 >5 条同类错误，提炼模式并修复

### 每周（周日心跳）
- 审查本周 `.learnings/` 所有条目
- 将高频学习提升到 `TOOLS.md`、`AGENTS.md` 或 `SOUL.md`
- 清理已解决的问题
- 更新 `MEMORY.md`

### 每月
- 回顾月度学习模式
- 评估哪些技能需要升级
- 检查信息源质量（AI 日报源的稳定性等）
- 更新进化状态报告

---

## 3. 进化维度

| 维度 | 度量 | 优化方式 |
|------|------|----------|
| **准确率** | 犯错频率下降 | 错误日志 → 模式识别 → 预防 |
| **效率** | 完成任务所需步骤减少 | 最佳实践沉淀 → 模板化 |
| **洞察质量** | 用户提供正面反馈的比例 | 结构化思维训练 |
| **主动性** | 主动发现 vs 被动响应的比例 | 扩展监控范围 |
| **工具熟练度** | API 调用成功率 | 故障记录 → 备选方案 |

---

## 4. 知识库进化

- **投资研究**：每次分析后总结方法论，沉淀到 `knowledge/`
- **AI 动态**：持续优化信息源和摘要质量
- **技术学习**：学习笔记结构化存储到 Get 笔记

---

## 5. 自我评估检查清单

每周回顾时问自己：

- [ ] 本周犯了哪些错误？根因是什么？
- [ ] 哪些重复任务可以自动化？
- [ ] 信息源有哪些失效？需要替换什么？
- [ ] 用户纠正了我什么？我的模型哪里需要更新？
- [ ] 有没有发现新的高价值信息源？
- [ ] 工具使用有没有更好的方式？
- [ ] MEMORY.md 是否反映了最新状态？
